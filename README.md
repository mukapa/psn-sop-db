# psn-sop-db

HWPX로 배포되는 재난안전통신망 SOP를 DB로 정리하는 방법을 연구합니다.

## 개요

재난안전통신망 표준운영절차(SOP)는 HWPX 형식으로 배포되며, 권(卷)마다 기관 명칭이 다르게 기록되는 등 활용에 어려움이 있습니다.  
이 프로젝트는 SOP를 PostgreSQL DB에 정규화하여 검색·활용이 용이하도록 변환하는 파이프라인을 제공합니다.  
소스코드는 LLM 바이브코딩으로 작성되었으며 MIT 라이선스를 적용합니다.

## 준비물

### 1. 표준운영절차 (SOP)
[재난안전통신망 종합지원시스템](https://www.safenet.go.kr/) 접속  
→ 정책지원 > 표준운영절차 게시판에서 최신 SOP 파일 다운로드

### 2. 기관명칭 정보
[행정표준코드관리시스템](https://www.code.go.kr/) 접속  
→ 검색항목 우측 하단 **"기관코드 전체자료"** 버튼 클릭  
→ 압축 해제 후 `기관코드 전체자료(유형분류 의미추가).txt` 저장

### 3. PostgreSQL
- 배포 소스코드의 DB 계정정보는 id : postgres // password : 1 로 설정되어 있습니다.
- 실제 운용 시 소스코드에서 비밀번호를 변경하여 사용하세요.

## 활용 순서

### Step 1. HWPX → DOCX 변환
내려받은 SOP 파일을 **모두 DOCX 형식으로 변환**하여 저장합니다.  
(한글 프로그램에서 일괄 변환 가능합니다. 공무원이라면 보통 설치되어 있을 겁니다...)

### Step 2. 입력 폴더 준비
`c:/input_docx` 폴더를 생성하고 변환된 DOCX 파일 5개를 모두 넣어주세요.  
(1권은 처리하지 않으므로 제외해도 됩니다.)

### Step 3. DOCX → YAML 변환
`c:/output_yaml` 폴더를 생성한 후, `release`에서 내려받은 `docx2yaml.py`를 실행합니다.

```bash
python docx2yaml.py
```

### Step 4. DB 테이블 생성
`release`에서 내려받은 `query.for.make.tables.sql`을 PostgreSQL에서 실행하여 테이블을 생성합니다.

### Step 4-1. DB 기초자료 입력
기관코드 전체자료(유형분류 의미추가).txt 를 org_master 테이블에 input 합니다. dbeaver 사용 기준으로 구분자는 '  '(따옴표 제외)를 복사해서 사용하고, 문자포맷은 EUC-KR 지정, NULL value make 에 대문자로 NULL 을 입력하면 됩니다. 

### Step 5. YAML → DB 입력
`yaml2sql.py`를 실행하여 YAML 데이터를 PostgreSQL 테이블에 입력합니다.

```bash
python yaml2sql.py
```

### Step 6. 결과 조회
목적에 맞는 쿼리 파일을 실행하면 정리된 SOP 데이터를 확인할 수 있습니다.

| 쿼리 파일 | 대상 |
|-----------|------|
| `output.query.for.police.sql` | 경찰관서 필터링 및 정렬 |
| `output.query.for.all.users.sql` | 전체 기관 출력 (기관명 보정 없음) |
